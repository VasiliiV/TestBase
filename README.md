Полностью разработан мной как полноценная учебная платформа для тестировщиков с UI, API, БД, моками, Docker и автотестами (UI, API, unit).

## Цель проекта:

Дать тестировщику реальную среду, в которой можно потренироваться:

- писать тест-кейсы и баг-репорты, взаимодействуя с живым UI
- изучать фронтенд + бекенд + база данных
- работать с API (GET/POST/PUT/DELETE)
- разбираться в аутентификации, токенах, ошибках
- запускать автотесты на Java и традиционные DevOps-вещи

Проект симулирует мини-продукт — в нём есть UI, API, DB, Docker, WireMock и автоматизация.

## Автотесты

В проекте используются три типа автотестов:

- Автотесты на **Java** — используются для тестирования API и бизнес-логики
- Unit-тесты фронтенда (**Jest**) — находятся в папке `__tests__`
- UI и API автотесты (**Playwright**) — находятся в папке `tests`

Unit-тесты и e2e / API автотесты запускаются раздельно и не конфликтуют между собой.
Для анализа результатов автотестов используется Allure-отчёт.

## Про запуск проекта:

### 1. Установка Node.js:
   Если у вас еще нет Node.js, скачайте его с официального сайта Node.js. Рекомендуется выбрать "Recommended For Most Users".
   Сайт: https://nodejs.org/en

### 2. Запуск проекта (два способа):
   1-й способ (через скрипт):
   Если у вас установлен Git Bash или WSL, запустите first-start.sh скрипт через терминал: ./first-start.sh.
   Если не активен скрипт, то прописать в терминале: chmod +x first-start.sh
   Важно: Если вы используете стандартный терминал Windows, скрипт .sh может не работать. В этом случае используйте 2-й способ.
   
   2-й способ (ручной запуск):
   Удалите папку node_modules в папке my-app.
   Откройте терминал в папке my-app 'cd my-app' и введите команду 'npm i' для установки зависимостей.
   Затем введите cd ./my-app, cp .env.example .env.local и npm run dev для запуска приложения.
